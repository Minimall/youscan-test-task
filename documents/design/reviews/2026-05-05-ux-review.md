# UX review: prototype.html
**Date:** 2026-05-05  
**Scope:** Full prototype — UX and usability, whole app  
**Method:** Live browser testing via Playwright MCP at 1440px and 375px

---

## Fixed during review

### Bug — `generateForecast` crash on new/preset rules
**File:** `prototype.html:770`  
**What:** `rule.id.charCodeAt(1)` threw `TypeError: Cannot read properties of undefined` whenever the rule editor was opened via a preset, because preset drafts and blank drafts have no `id` field.  
**Fix:** `rule.id?.charCodeAt?.(1) || rule.name?.charCodeAt?.(0) || 7`

### Critical — Demo controls FAB overlapped "Edit rule" button
**File:** `prototype.html:403` (`#debug` CSS)  
**What:** The floating "Demo controls" pill (`position:fixed; bottom:20px; right:20px`) sat directly over the "Edit rule" button in the detail pane footer. At 1440px the FAB was `left:1264–right:1420`, the Edit rule button `left:1334–right:1420` — exact overlap. Also blocked mobile action buttons at 375px.  
**Fix:** Changed `right:20px` → `left:20px`. FAB now sits in the bottom-left, clear of all product UI.

### High — Threshold field split "1 hour" across lines in panel mode
**File:** `prototype.html:1481` (`ruleFormFieldsHTML`)  
**What:** In 440px panel mode, the threshold row `flex-wrap:wrap` split "Trigger when more than [3] mentions in [1]" on line 1 and "[hour ▾]" alone on line 2 — the time unit lost its context.  
**Fix:** Wrapped "mentions in [N] [unit]" in a `display:inline-flex; flex-shrink:0` span so the time-window group never splits.

### Medium — Rule editor double-header
**File:** `prototype.html:1345` (`viewRuleEditor` sub-bar)  
**What:** The topbar breadcrumb showed "Alerts > Alert Rules > New rule" while the content sub-bar had its own "Cancel › [rule name]" pattern — two simultaneous back-navigation affordances pointing to the same destination but styled differently.  
**Fix:** Removed "Cancel ×" button and "›" separator from the sub-bar. Sub-bar now shows only the rule name as title + "× Cancel" + "✓ Save rule" actions on the right.

### Medium — Rules list "Created by" column took space needed by action icons
**File:** `prototype.html:1249, 1275` (`viewRules` table)  
**What:** The "Created by" column (not in MVP spec) occupied ~15% of table width, causing hover-reveal action icons to appear crammed into a `1px` column immediately after it — visually ambiguous (icons appeared to belong to the person name, not the rule).  
**Fix:** Removed `<th>Created by</th>` header and `<td>${r.createdBy}</td>` cells. The actions column now has natural room to breathe.
